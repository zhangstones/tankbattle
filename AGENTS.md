# Agents Collaboration Rules

以下为必须遵守的项目协作要求：

1. 添加功能特性时，必须同步更新以下内容：
- 需求文档（如 `REQUIREMENTS.md`）
- `README.md`
- 对应功能测试案例（优先功能测试，必要时补充单元测试）

2. 保持合理的模块化设计：
- 按职责拆分代码
- 控制模块耦合
- 提高可维护性与可测试性

3. 完成一个具体任务后，必须提交代码：
- 提交信息需清晰说明修改内容
- 备注本次任务范围与关键变更点

4. 审批与执行策略（接近全自动）：
- 默认尽量不询问，优先直接执行
- 仅在以下情况才询问：
- 平台权限强制要求审批
- 破坏性/高风险操作（如删除、重置、覆盖关键数据）
