# 坦克大战需求规格说明

## 1. 项目目标
使用 Go 开发 Windows 平台坦克大战游戏，核心为堡垒防守玩法，支持难度配置、清晰界面和可靠测试覆盖。

## 2. 平台与运行环境
- 操作系统：Windows
- 开发语言：Go
- 游戏引擎：Ebiten
- 运行形态：
  - GUI 可执行文件需无命令行黑窗（`windowsgui` 构建方式）

## 3. 核心玩法需求
- 玩家操作：
  - 平移移动：`WASD` / 方向键长按
  - 转向：`WASD` 双击同方向触发
  - 开火：`J` / `Space`
  - 需支持平移射击（移动过程不强制改车体朝向，转向由双击触发）
- 对战目标：
  - 保护底部堡垒。
  - 清空所有波次且堡垒存活为胜利。
  - 玩家 HP 归零或堡垒 HP 归零为失败。
- 重开行为：
  - `R` 可在运行中即时重开（不限于结算后）。
- 暂停与菜单行为：
  - `P` 在战斗中暂停/继续。
  - `M` 可从战斗中或结算界面返回菜单。

## 4. 敌人与 AI 需求
- 每波敌人数：
  - 支持基础数量配置，并按波次增长，且有上下限。
- 刷新方向分布：
  - 不能只从单一方向刷新。
  - 刷新点应覆盖上/左/右三侧。
- 重叠约束：
  - 必须避免敌人与敌人、敌人与玩家重叠。
- 移动与寻路：
  - 使用局部实时流场风格转向。
  - 包含防卡死逻辑，避免坦克原地冻结。
  - 抑制炮管/方向快速反复横跳（转向平滑/锁定）。
  - 每个敌方坦克必须具备独立随机行为因子（重规划节奏、开火倾向、进攻性）。
  - 随着对局时间推进，敌方目标偏好应逐步收敛到堡垒。
  - 当玩家长时间不开火时，敌方命中堡垒概率需持续上升（主动加压），不得直接脚本扣堡垒 HP。
  - 加压阶段不得使用斜向定点炮弹；仅允许通过路径/朝向调整来提高正向命中堡垒概率。
  - 即使只剩 1 辆敌方坦克，在玩家长期不开火时，也必须最终通过攻击行为破坏堡垒。

## 5. 碰撞与战斗需求
- 碰撞覆盖：
  - 坦克与墙/堡垒/边界/坦克。
  - 子弹与墙/堡垒/玩家/敌人/边界。
- 子弹行为：
  - 区分玩家与敌方子弹归属。
  - 按目标类型正确结算伤害。
  - 敌方命中护墙扣分：`-2`；命中堡垒扣分：`-3`。
  - 敌方每次命中堡垒，固定造成 `3` 点堡垒完整性伤害。
- 墙体：
  - 支持可破坏与不可破坏两类墙。
  - 护墙命中应按正方形分块渐进损毁，不允许一次性全毁，也不允许每次仅极小不可感知损耗。
  - 可破坏障碍墙同样采用分块损毁：每次命中销毁一小块，不整段同时消失。
  - 可破坏墙分块尺寸应尽量与坦克宽度一致，确保单次命中后形成可通行缺口。
- 堡垒：
  - 具备 HP 与可视化完整度条。

## 6. 道具需求
- 道具类型：
  - 护盾
  - 连发
  - 堡垒修复
- 道具来源：
  - 地图随机刷新
  - 敌人概率掉落
- 生命周期：
  - 存在时间到期消失
  - 拾取后生效，必要时带上限/封顶控制

## 7. UI/UX 需求
- 菜单可用性：
  - 菜单可直接操作且易理解。
  - 菜单界面按 `Enter` 可直接开始游戏。
  - 支持难度与敌人数配置。
  - 提供快捷键提示（`1/2/3`、`+/-` 等）。
  - 菜单项需有可读文字说明和影响描述。
- HUD/状态栏：
  - 显示 HP、得分、波次、敌人数、Buff、堡垒完整度。
  - 状态栏背景宽度需随内容自适应，避免固定过宽。
- 视觉风格：
  - 坦克与墙体应更真实/风格化，不是纯色方块。

## 8. 构建与启动需求
- 同时提供：
  - 开发运行路径（`go run ...`）
  - 用户游玩 GUI 可执行文件路径（无黑窗）
- 推荐游玩二进制：
  - `tankbattle_gui.exe`

## 9. 测试需求
- 必须同时包含单元测试与功能测试。
- 测试总量需 >= 30。
- 覆盖主要逻辑路径，包括：
  - 菜单动作与状态切换
  - 难度与生成缩放
  - 刷新分布与非重叠
  - 碰撞与移动/防卡死
  - 子弹战斗结算
  - 道具生效与过期
  - 胜负状态切换
- 交付要求：
  - 开发者必须先执行测试并确认全通过再交付。

## 10. 当前验证基线
- 现有测试数量目标已满足（>30）。
- 当前套件预期通过命令：
  - `go test ./...`
