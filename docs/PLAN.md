# 背景简化方案计划

## 1. 目标

当前技术实现不适合继续追求复杂背景重设计。
本轮目标调整为：

- 用低风险、低复杂度方案，把背景稳定收敛到“地面坦克战场”语义。
- 背景只提供基础氛围，不承担复杂美术表达。
- 优先保证主体可读性、快照稳定性和实现成本可控。

## 2. 采用方案

采用“纯色背景 + 弱网格 + 少量沙石模拟”方案。
具体要求：

- 纯色地表底色为主，不再追求复杂地貌分层。
- 保留很弱的地表网格，作为战场秩序感来源。
- 只允许少量沙石、尘土点或轻微地表噪声作为点缀。
- 禁止使用大面积强辉光、漂浮亮点、强扫描线、雷达感线条。
- 禁止继续堆叠复杂战壕、防线、推进通路和大构图装饰。

## 3. 设计取舍

放弃复杂方案的原因：

- 当前实现手段更适合稳定的程序化简背景，不适合高完成度美术场景。
- 复杂背景容易把主体压暗、压脏，反而削弱战斗可读性。
- 复杂构图会显著提高 golden 维护成本和界面回归风险。

保留简化方案的收益：

- 题材识别仍然正确。
- 视觉更干净，HUD、坦克、子弹和堡垒更容易看清。
- 实现简单、风险低、易于长期维护。
