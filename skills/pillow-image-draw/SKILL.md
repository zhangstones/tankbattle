# Pillow ImageDraw

## 目标

- 用代码稳定迭代图形，而不是每轮重写绘制逻辑。
- 让视觉修改可控、可回退、可验证。

## 原则

1. 先定结构，再调细节
- 先明确背景、主体、辅助元素的分层和关系。
- 先确认大构图，再调整长度、角度、间距、透明度。

2. 一次只改一个维度
- 单次迭代只改大小、位置、角度、颜色、装饰之一。
- 避免同时修改构图和风格。

3. 用统一几何规则约束相关元素
- 相关元素应共用锚点、方向向量或主轴。
- 不要让相互关联的部件各自独立漂移。

4. 主体优先于辅助元素
- 光圈、边框、背景只服务主体，不抢中心。
- 辅助元素显得多余时，优先删减而不是继续叠加。

5. 小尺寸可读性优先
- 图标最终会缩到小尺寸显示。
- 过细细节、过近切线、过多同权重装饰，都会降低识别度。

6. 保持增量修改
- 优先在现有脚本上局部调整。
- 只有当当前结构已经阻碍迭代时，才重构。

## 预览规则

1. 重要变更优先输出到新文件名，避免缓存误判。
2. 预览与代码不一致时，先查脚本内容，再查输出路径，最后查查看工具。
3. 定稿前不要急着同步到正式资源链。

## 常见判断

- “没变大/没变小”：
先检查主轮廓是否变化，而不是只改内部元素。

- “角度不对”：
先统一主轴，不要只改单个局部零件。

- “不要露出来”：
优先收短插入段或调整锚点，而不是整体缩放全部元素。

- “装饰看着怪”：
优先减层，避免继续叠加效果。

## 收尾

1. 定稿后只保留最终资产。
2. 放弃程序绘图路线时，删除实验脚本，避免后续误用。
