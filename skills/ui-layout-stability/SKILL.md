# HUD 布局稳定性技能（精简版）

## 适用场景

- HUD/菜单在持续迭代中出现对齐抖动、层级混乱或视觉负担上升。
- 多轮微调后，信息位置频繁变化导致玩家认知成本增加。
- 需要将一次设计优化沉淀为可复用的布局原则。

## 设计原则

1. 信息分层
- 先区分核心战斗信息、次要状态信息、操作提示信息。
- 核心信息放高频视线区，次要信息放边缘区域。

2. 分区稳定
- 左右 HUD 分工固定，不在迭代中频繁换区。
- 同类信息长期保持同位置，降低学习和记忆成本。

3. 对齐一致
- 使用统一网格、列宽和行高管理 label/value。
- 避免局部临时偏移，所有文本对齐应由统一规则驱动。

4. 主次清晰
- 通过字重、颜色、透明度表达优先级，不靠堆砌元素。
- 边框与背景弱化处理，优先保障可读性和信息扫描效率。

5. 迭代可控
- 每次只改一个维度（布局、样式、文案三选一）。
- 采用“修改 -> 构建 -> 运行观察 -> 确认”的短闭环，减少返工。

6. 交付收敛
- 最终仅保留已确认方案，清理试验性分支代码。
- 文档记录目标与原则，不沉淀像素级过程细节。

## 验收清单（原则级）

- 信息层级能快速识别，核心信息无干扰。
- 同类信息位置稳定，布局未出现明显跳变。
- label/value 对齐一致，扫描路径连贯。
- HUD 视觉负担可控，无不必要装饰噪声。
- 代码中无遗留实验分支或冲突设计实现。
